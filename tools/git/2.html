# 1. 配置deploy.sh

在根目录创建文件:deploy.sh

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.yourwebsite.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果你想要部署到 https://USERNAME.github.io
# longweixia分别是账户名和项目名 master是推送的分支
git push -f https://github.com/longweixia/longweixia.git master  

# 如果发布到 https://USERNAME.github.io/<REPO>  REPO=github上的项目
# git push -f git@github.com:USERNAME/<REPO>.git master:gh-pages

cd -

# 2. 配置config.js

docs/.vuepress/config.js

base项需要匹配要发布的项目名，如：base: '/longweixia/',

# 3. 去github上配置page

进入项目（longweixia）的主页面，上面的tab页的最右侧有个settings，点击进入

然后找到左侧的page菜单，进入，然后配置Source项目，分支选默认的master，Select folder选/(root)

配置好后点击保存，可以看到上面显示了配置的page站点，之后就可以用这个页面来访问了(https://longweixia.github.io/longweixia/)

# 4. 现在去提交代码

git add .

git commit -m '提交内容'

git push

提交好后，再输入命令:bash deploy.sh去发布

这时候可能出现无法识别bash指令，没关系，我们用git工具的命令行来操作，找到我们的项目文件夹，右键git bash，然后输入bash deploy.sh

如果此时没问题的话，窗口最后会输出:

To https://github.com/longweixia/longweixia.git
 + ddd5c08...43dff02 master -> master (forced update)

此时文章就更新了，我们去对应的站点查看。

但是不巧的是很多人这个节点会报错

# 5. 报错处理

参考地址 (opens new window)

如果报这个

git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.

说明公钥出问题了,需要删除.ssh下文件,然后重设置用户名和邮箱再重新生成ssh公钥即可解决

第一步：

进入C:\Users\user\.ssh删除.ssh下所有所有文件。

第二步：

1.设置用户名

git config --global user.name 'longweixia'

2.设置用户名邮箱

git config --global user.email '481033454@qq.com'

3.查看设置

git config --list

可以看到配置成功了。

4.然后继续输入命令:修改后面的邮箱即可

ssh-keygen -t rsa -C "481033454@qq.com"

接着一路回车

然后去github上配置

进入个人中心settings，进入左侧的菜单SSH and GPG keys。

点击New SSH keys，然后title输入rsa，key需要去找本地文件的对应的值（找到C:\Users\user\.ssh\id_rsa.pub，复制里面的值，然后输入到key里面。）

接着再bash deploy.sh就没有问题了
